    def load_payment_transactions()  # Load & validate

Design principles: Modular ‚Ä¢ Defensive ‚Ä¢ Observable ‚Ä¢ Idempotent

Join Strategy

Payments (base table)
    LEFT JOIN Accounts AS src  ON src_account_id
    LEFT JOIN Accounts AS dest ON dest_account_id
    LEFT JOIN RiskFeed AS src  ON src_account_id
    LEFT JOIN RiskFeed AS dest ON dest_account_id
    LEFT JOIN FraudLabels      ON payment_id
Why LEFT Joins?
Never lose payment transactions
Surface data quality issues
Fraud labels arrive late
Default Values
risk_flag = 0 (clean)
fraud_flag = False (unlabeled)
fraud_type = None
